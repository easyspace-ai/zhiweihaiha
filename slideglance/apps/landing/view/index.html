Viewer — why use it

  • Local-first

    Files never leave your tab, process, or worker. No upload, no server, no telemetry path for the deck contents.

  • Deterministic

    Same input plus pinned fonts produce byte-identical SVG and pixel-identical PNG across machines.

  • Offline-capable

    Once the WebAssembly bundle is loaded, the viewer needs zero network access — open decks on a plane.

  • Selectable text

    Text-mode SVG keeps copy / search / accessibility intact; path-mode glyphs are available for resvg-rasterized PNG.

  • Wide PPTX coverage

    Theme inheritance, tables, charts, gradients, EMF-wrapped bitmaps, WordArt warps, transitions, notes, sections, embedded fonts.

  • Multi-surface

    Same Rust + WASM core powers the Chrome extension, web playground, desktop viewer, embeddable React / Vue / Web Component, and a native CLI.

  • Font fallback chain

    Authored fonts are matched against installed faces; every substitution is surfaced in the UI so silent visual drift never ships unnoticed.

  • Embeddable

    @slideglance/viewer is a React shell that drops into any page, Storybook story, or doc site. Same component the playground and Chrome extension mount.

  • Native CLI

    slideglance convert deck.pptx --output out/ for batch SVG / PNG export from a build pipeline. Deterministic, no Node runtime needed.
