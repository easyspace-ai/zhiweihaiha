Build

Author .sgx, compile to .pptx

@slideglance/builder compiles a tiny XML DSL into deterministic PPTX. Layout, styling, and validation live in the source file — diffable in git, reviewable on a pull request, free of "I'll fix it in the GUI".

Get the VS Code extension → Browse the XML reference → npm install @slideglance/builder

Example source

<SlideGlance xmlns="urn:slideglance:builder:v1"
             xsi:schemaLocation="urn:slideglance:builder:v1
               https://unpkg.com/@slideglance/builder@^0.1/builder.xsd">
  <Document size="16:9" />

  <Styles>
    <Style name="kicker"   fontSize="14" bold="true" color="666" />
    <Style name="title-md" fontSize="32" bold="true" />
  </Styles>

  <Slide>
    <VStack padding="64" gap="24">
      <Text class="kicker">Q4 · UPDATE</Text>
      <Text class="title-md">Highlights</Text>
      <Foreach items='[{"label":"MRR","value":"$420K"},
                       {"label":"NRR","value":"118%"}]' as="m">
        <HStack gap="8">
          <Text class="kicker">{m.label}</Text>
          <Text fontSize="28" bold="true">{m.value}</Text>
        </HStack>
      </Foreach>
    </VStack>
  </Slide>
</SlideGlance>
1 · Install 
npm install @slideglance/builder
# or: pnpm add @slideglance/builder
2 · Compile 
import { buildPptx } from "@slideglance/builder";
import { readFileSync, writeFileSync } from "node:fs";

const xml = readFileSync("deck.sgx", "utf8");
const { pptx, diagnostics } = await buildPptx(
  xml,
  { w: 1280, h: 720 },
  { lint: { enabled: true } },
);

const bytes = await pptx.write({ outputType: "uint8array" });
writeFileSync("deck.pptx", bytes);
console.log(diagnostics);

Building blocks

The DSL surface fits on a postcard: six element families, each with a job. Mix-and-match for any deck shape.

01

Document & Slides → 

<Document size="16:9" defaultMaster="body" />
<Slide>
  <VStack padding="64">
    <Text fontSize="32" bold="true">Hello</Text>
  </VStack>
</Slide>

<Document> sets canvas size + default master. Each <Slide> holds one root container. size accepts "16:9", "4:3", "A4", or explicit w/h.

Reference: <SlideGlance>, <Document>, <Slide>, <Fragment>

02

Stacks & layers → 

<HStack gap="24" alignItems="center">
  <VStack gap="8" w="50%">
    <Text class="kicker">A</Text>
  </VStack>
  <VStack gap="8" w="max">...</VStack>
</HStack>
<Layer><Image x="40" y="40" .../></Layer>

<VStack> / <HStack> stack along one axis with gap, padding, alignItems. w="max" claims slack; <Layer> opts into absolute positioning.

Reference: <VStack>, <HStack>, <Layer>

03

Text & inline formatting → 

<Text fontSize="16" lineHeight="1.4">
  Sales rose <B>23%</B> in
  <Span color="00aa66">Q4 2025</Span>.
</Text>
<Ul><Li>item</Li></Ul>

<Text> supports <B> / <I> / <U> / <S> / <Mark> / <Span> / <A> for inline runs. <Ul> / <Ol> handle bulleted + numbered lists.

Reference: <Text>, <Ul>, <Ol>

04

Styles — tokens → 

<Styles>
  <Style name="kicker" fontSize="14" bold="true"
         color="666666" />
  <Style name="title"  fontSize="32" bold="true" />
</Styles>

<Text class="kicker">METRIC</Text>
<Text class="title">$420K MRR</Text>

Declare once in <Styles>, apply via class="...". Multiple classes combine left-to-right; inline attrs always win.

Reference: <Styles>, <Style>

05

Templates & Use → 

<Template name="metric-card">
  <VStack class="card" padding="24" gap="8">
    <Text class="kicker">{label}</Text>
    <Text class="title">{value}</Text>
    <Slot name="default" />
  </VStack>
</Template>

<Use template="metric-card" label="MRR" value="$420K">
  <Text>+12% QoQ</Text>
</Use>

Extract a shape once, reuse with <Use template="...">. Parameters interpolate via {name}; <Slot> carries multi-element content.

Reference: <Templates>, <Template>, <Use>, <Slot>

06

Foreach, Import & Master → 

<Import src="styles/typography.xml" />

<Master name="body">
  <MasterText x="48" y="36" fontSize="10">
    Q4 · UPDATE
  </MasterText>
</Master>

<Foreach items='[
  {"label":"MRR","value":"$420K"},
  {"label":"NRR","value":"118%"}
]' as="m">
  <Use template="metric-card"
       label="{m.label}" value="{m.value}" />
</Foreach>

<Foreach> drives lists from JSON, <Import> splits decks across files, <Master> holds the chrome (header / footer / page number) every slide shares.

Reference: <Foreach>, <Import>, <Document>

Browse the full XML reference → Every node, attribute, type, and child relation is generated from the same compiled registry that drives the parser, XSD, and JSON Schema. Prefer raw markdown? reference.md still ships on every release.

  • Flex-style layout primitives

    <VStack>, <HStack>, <Layer>, and a yoga-layout engine do the positioning. Padding, gap, alignment, intrinsic sizing — all CSS-flexbox-shaped without the CSS.

  • Templates + Foreach

    Extract repeating shapes into <Template> and call them with <Use>. Drive list-shaped slides from a JSON array via <Foreach> — N near-identical slides become one loop.

  • Master + Styles + Imports

    Chrome lives in <Master>, type ramps in <Styles>, and <Import> pulls shared chapters across decks. Rebranding is a 60-second token change.

  • autoFit + opentype measurement

    Layout uses real font metrics (Pretendard / Noto Sans JP + optional caller-supplied TTFs) so wrap decisions match the renderer. Overflow triggers font / gap / scale fallbacks automatically.

  • XSD + JSON Schema

    Every node and attribute lives in a versioned schema at unpkg.com/@slideglance/builder@^0.1/builder.xsd. Drop it into the Red Hat XML extension for autocomplete and on-save validation in any IDE.

  • 30-rule lint pass

    Catches horizontal / vertical overflow, low contrast, mixed baselines in rows, unused styles, hardcoded hex, accessibility gaps, and slide-count regressions. Runs on every build and on save in the VS Code extension.

Sample lint output 
[warn] TEXT_OVERFLOW_H  Slide 6: <Text> natural single-line width ~547px exceeds box width 533px. Glyphs spill past the right edge.
[error] OUT_OF_PARENT Slide 3: <VStack> overflows its hstack parent by 12px. Widen the parent or shrink the child.
[info] LOW_CONTRAST   Slide 2: text #6E5A45 on #FFFBF5 → contrast 4.21, below WCAG AA 4.5.
[info] UNUSED_STYLE   Deck declares <Style name="callout-dim" /> but no element references it.

VS Code extension

Slide Builder registers the .sgx schema, opens a live-preview pane next to the source, and runs the same lint set on save. Click any shape in the preview to jump back to the line that produced it.

  • Live PPTX preview (incremental — survives edit cycles).
  • Per-slide cache invalidation, fresh remount on file swap.
  • One-click .pptx export from the toolbar.
  • Schema-aware autocomplete via the Red Hat XML extension.

Programmatic API

@slideglance/builder on npm exposes buildPptx(xml, slideSize, options) — pure async TypeScript, no native deps, runs in Node and browsers. Wire it into a CI step, a CMS publish hook, or a "render decks from a spreadsheet" script.

  • Returns a pptxgenjs instance + structured diagnostics + source map.
  • Same lint catalog as the VS Code preview — fail your build on errors.
  • Inject custom fonts; render off the bundled Pretendard / Noto Sans JP.
